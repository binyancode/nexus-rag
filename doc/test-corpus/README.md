## 为什么这样选（对齐设计的可测性）

- **覆盖三个申报大类**，每类至少 3 篇。
- **产生多类实体**：`AppType`（IIT/IND/NDA）、`Reg`（各法规）、`Category`（伦理、疫苗、细胞治疗、注册核查…）。
- **产生多类边**：
  - `requires`：IIT → 伦理审查办法 / 人类遗传资源条例 / 临床研究管理办法；IND、NDA → 药品管理法 / 药品注册管理办法 / 实施条例；NDA → 疫苗管理法。
  - `references`（文本内引用，抽边样本丰富）：
    - 药品注册管理办法「根据《药品管理法》《中医药法》《疫苗管理法》《行政许可法》《药品管理法实施条例》…制定本办法」；正文多处引用《药品管理法》《疫苗管理法》具体条款。
    - 药品管理法实施条例「根据《药品管理法》…制定本条例」。
    - 疫苗管理法「本法未作规定的，适用《药品管理法》《传染病防治法》…」。
    - 伦理审查办法「依据《民法典》《生物安全法》《人类遗传资源管理条例》…」。
    - 临床研究管理办法「根据《药品管理法》《医疗器械监督管理条例》《涉及人的生命科学和医学研究伦理审查办法》…」，附则引用《干细胞临床研究管理办法（试行）》。
  - `belongs_to`：疫苗管理法 → 预防用生物制品/疫苗分类。
  - `evidence`：概念 ↔ 具体条文块（fullname 溯源）。
- **可复现集合运算示例**：药品管理法 / 药品注册管理办法 / 实施条例同属 IND 与 NDA（交集样例）；疫苗管理法仅 NDA、三部 IIT 文档仅 IIT（差集样例）。
